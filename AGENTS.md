# AGENTS.md

챠이로 쿠냐 팬 사이트에서 작업하는 코딩 에이전트를 위한 지침. **이 파일이 정본이고**
`CLAUDE.md` · `GEMINI.md` 는 이 파일을 import 한다. 규칙이 바뀌면 여기만 고친다.

사람이 읽을 프로젝트 개요는 `README.md` 에 있다.

## 이 저장소가 무엇인가

버추얼 스트리머 **챠이로 쿠냐**의 비공식 팬 사이트. 정적 HTML/CSS/JS 이며
**빌드 과정도 패키지 매니저도 없다.** 저장소 루트를 GitHub Pages 가 그대로 서빙한다.

- 배포: <https://chnu-kim.github.io/chyaro-kunya/> (origin `chnu-kim/chyaro-kunya`, HTTPS)
- 로컬 확인: `python3 -m http.server` 후 <http://localhost:8000>
  `file://` 로 열면 절대 URL 에셋과 폰트 로드가 실제 배포와 달라진다.
- 테스트·린트·CI 없음. 검증은 브라우저에서 눈으로, 배포 후엔 실제 URL 로 한다.

## 화면

| 파일 | 내용 |
|---|---|
| `index.html` | 홈 · 두 화면으로 가는 런처 |
| `landing.html` | 소개 — 프로필, 주요 컨텐츠, 채널 링크 |

`css/site.css` 는 세 페이지가 공유하는 토큰 + 크롬(nav·푸터·테마 토글).
`js/site.js` 는 테마 토글·테마별 이미지 스왑·푸터 연도. `js/games.js` 는 게임 보드.

## 절대 규칙

### 1. 디자인 토큰이 색·타입의 정본이다

`css/site.css` 의 `:root` 토큰을 `var(--token)` 으로만 참조한다. **생 hex 를 새로
쓰지 않는다.** 라이트/다크 두 테마가 모두 살아 있어서, 토큰을 우회하면 한쪽 테마에서
조용히 깨진다. 브랜드 토큰에서 값을 파생해야 하면 `color-mix(in srgb, …)` 를 쓴다.

핑크 액센트(`--accent`)는 **화면당 최대 2회.** 배경 워시로 쓰지 않는다.
중립(종이·잉크·브라운)이 화면의 80% 이상을 차지해야 한다.

### 2. 채널은 3개뿐 — 디스코드는 없다

치지직 · 유튜브 · X. **디스코드는 운영하지 않으므로 링크·아이콘·언급을 만들지 않는다.**
과거 대화나 디자인 시스템 문서(`DESIGN.md`, `SKILL.md`)에 "치지직·유튜브·X·디스코드"라고
적혀 있어도 그건 틀린 기록이다. 실제로 이 사이트엔 "Discord · 연결 예정" 플레이스홀더가
살아 있었고 사용자가 직접 지우게 했다. 채널이 늘면 사용자가 알려준다 — 추측해서 넣지 않는다.

```
치지직  https://chzzk.naver.com/97c69a12835bba786fb82f2c849f2ba8   (URL이 해시라 표시할 핸들 없음)
유튜브  https://www.youtube.com/@CHYAILOKUNYA
X       https://x.com/chyailokunya
```

### 3. 이미지는 사용자가 제공한다

사이트에 들어갈 이미지(쿠냐 단독, 쿠냥이, 키 비주얼 등)는 **사용자가 직접 준비해서
전달한다.** 생성하지 말고, 필요한 목록을 명확히 정리해 요청한다.

반대로 **장식 요소(고양이 발바닥·고양이·리본·아이콘)는 이미지가 아니라 인라인 SVG 로
직접 작성한다.** 이모지 아이콘(✨🚀🎯)은 금지 — 미니멀 블랙 모티프 SVG 를 쓴다.

`assets/kunya-debut-336.jpg` 같은 파생본은 원본에서 `sips` 로 만든 것이다
(macOS 기본 도구, WebP 는 못 만든다).

### 4. 공개 저장소 — 푸시 전에 무엇이 나가는지 확인한다

푸시는 되돌릴 수 없다. 지워도 git 히스토리에 남고 캐시·인덱싱된다.

`.gitignore` 가 이미 막고 있는 것: 작업용 스크래치 이미지(`mq*-image.png`),
Open Design 내부 메타데이터(`*.artifact.json`, `.od-skills/`), 미참조 pdf.
특히 `mqx0rv5y-Gemini_Generated_Image_…-Photoroom.png` 는 **파일명 자체가 어떤 생성
도구를 썼는지 광고**한다.

새 파일을 커밋하기 전에 `git status` 로 스테이징을 확인하고, 푸시 후엔 추측하지 말고
**실제 배포 URL 을 찔러 404 를 확인**한다.

## 이 코드베이스에서 실제로 밟은 지뢰

새 코드를 쓸 때 같은 실수를 반복하지 않기 위한 목록이다. 전부 여기서 실제로 터졌다.

- **SVG 의 기본 `fill` 은 `currentColor` 가 아니라 검정이다.** `color` 만 지정하면
  stroke 는 테마를 따라가는데 fill 은 검정에 박혀서, 다크 배경에서 눈·코가 사라진다.
  SVG 마크에는 `fill: currentColor` 를 명시한다. 이 버그 하나가 카오모지·반전 테스트·
  다크 도장 세 곳을 동시에 조용히 깨뜨렸다.
- **`<a>`/`<button>` 에 `role="listitem"` 같은 비인터랙티브 role 을 붙이지 않는다.**
  link role 이 덮여 스크린리더가 링크로 announce 하지 않는다. 시각적으로 멀쩡하고
  클릭도 되기 때문에 리뷰에서 놓치기 쉽다. 목록형 링크는 네이티브 `<ul>`/`<li>` 로 감싼다.
- **라틴 전용 폰트에는 한글 페이스를 폴백으로 붙인다.** Gloock·Sacramento 는 한글
  글리프가 없어서, 스택에 한글 페이스가 없으면 모든 한글 제목이 OS 임의 폰트로 렌더된다.
  `--font-display` 는 `Gloock, 'Nanum Myeongjo', …`, `--font-script` 는
  `Sacramento, 'Nanum Pen Script', …` 로 글리프 단위 폴백을 만든다.
- **`font-display: swap` 은 레이아웃 폭을 일시적으로 부풀린다.** 한글이 폴백으로
  전각 시스템 폰트(~1.0em/자)에 떨어지면 Nanum Pen Script(~0.55em/자)보다 훨씬 넓어진다.
  nav 가로 넘침이 여기서 났다 — 좁은 화면 폭 계산은 폴백 폭 기준으로 해야 한다.
- **flex 자식은 `min-width: auto` 라서 콘텐츠 폭 아래로 줄지 않는다.** 넘침을 막으려면
  `min-width: 0` 을 명시한다.
- **"내용은 읽히는데 대상은 안 보이는" 버그.** 팔레트 스와치·반전 필드처럼 *테두리가
  없는* 요소는 배경과 같은 색이어도 안의 텍스트가 읽혀서 정상처럼 보인다. 실제로
  1.00:1 대비로 배포될 뻔했다. 테마를 따라가는 토큰으로 테두리를 주면 반대 테마에서
  똑같이 사라지므로, `color-mix(in srgb, var(--brand-ink) 55%, var(--brand-paper))`
  처럼 **테마 독립적인 값**을 쓴다.
- **`og:image` 는 반드시 절대 URL.** X·Slack 은 상대 경로를 무시해서 이미지 없는
  카드가 나간다. 저장소 이름을 바꾸거나 커스텀 도메인을 붙이면 세 페이지의
  `og:image`/`og:url` 을 같이 옮겨야 한다. `landing.html` 은 `og:type` 이
  `website` 가 아니라 `profile` 이라 앵커가 다르니 별도로 확인한다.
- **`gh` 계정 전환은 SSH 키 신원을 바꾸지 않는다.** 이 머신엔 GitHub 계정이 둘
  로그인돼 있고, 저장소 소유자(`chnu-kim`)가 SSH 키 주인이 아니다. remote 가 SSH 면
  `gh` 활성 계정을 chnu-kim 으로 맞춰도 푸시는 키 주인 명의로 나가 거부된다.
  `gh auth status` 로 활성 계정을 확인하되 그걸 믿지 말고, remote 를 HTTPS 로 두고
  `git -c credential.helper='!gh auth git-credential' push` 처럼 **해당 푸시에만**
  자격증명을 적용한다 — 전역 git 설정은 건드리지 않는다.

## 접근성 기준 (협상 대상 아님)

- 대비: 본문 4.5:1, 큰 텍스트/UI 컴포넌트 3:1. 눈으로 판단하지 말고 계산한다.
- 터치 타깃 44×44 이상. `.slink`·`.nav__link`·`.theme-toggle` 이 이 값을 지키고 있다.
- 포커스 링은 항상 보이게: `--focus` 2px. `--accent` 는 채움용이라 링으로 쓰면 대비가 안 난다.
- 장식(테이프·클립·리본·마크)은 전부 `aria-hidden="true"`.
- 새 창으로 열리는 링크엔 `<span class="sr-only">(새 창에서 열림)</span>`.
- `prefers-reduced-motion` 가드는 이미 있다 — 새 애니메이션도 그 안에 들어가야 한다.

## 코드 컨벤션

- **주석은 제약을 적는다.** 이 저장소의 주석은 "다음 줄이 뭘 하는지"가 아니라
  "왜 이 값이어야 하는지"를 적는다(`--nav-h` 유도 과정, `swapImages` 가 CSS
  `content:url()` 을 안 쓰는 이유). 그 톤을 따른다.
- **localStorage 는 신뢰하지 않는 입력이다.** `games.js` 의 `coerce`/`load` 를 보라 —
  파싱 성공 ≠ 배열, 대괄호 조회는 프로토타입 체인을 탄다(`status='constructor'`).
  빈 배열은 손상이 아니라 "사용자가 다 지웠다"이므로 시드로 되살리면 안 된다.
- JS 는 ES5 스타일 IIFE, 의존성 없음. 프레임워크를 새로 들이지 않는다.
- 사용자가 가리킬 요소(섹션·제목·CTA·반복 카드)에 `data-od-id="kebab-case"` 를 붙인다.
- 사용자 노출 문구는 전부 한국어. 톤은 다정하고 장난기 있되 과하지 않게.
  고양이 말투("냐")는 화면당 1회까지.
- 파일당 ~1000줄 이하. 큰 개편 전엔 `-v2` 로 복사한다.

## 알려진 미완

- `assets/og-cover.jpg` 는 `og-cover.html` 을 렌더한 결과가 **아니다.** 데뷔 사진을
  1200×630 에 패딩한 것이라 조판과 도장 워터마크가 빠져 있다. 이 런타임엔 렌더 경로가
  없어서 사용자가 브라우저로 캡처해 줘야 교체된다.
- `index.html`(카드 2장 런처)과 `landing.html`(실제 소개)의 정보 구조가 중복이다.
  합칠지 유지할지는 사용자 결정 대기 중 — 임의로 정리하지 않는다.
